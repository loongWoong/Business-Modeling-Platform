# 快速启动指南

## 🚀 启动步骤

### 1. 启动后端服务

打开第一个终端窗口：

```bash
cd backend_ddd
python run.py
```

**验证**：访问 `http://localhost:5000/health` 应该返回 `{"status": "healthy"}`

### 2. 启动前端服务

打开第二个终端窗口：

```bash
cd frontend_ddd
npm run dev
```

**验证**：浏览器自动打开 `http://localhost:3000`，应该能看到首页

### 3. 清除浏览器缓存

在浏览器中：
- 按 `Ctrl+Shift+R` (Windows) 或 `Cmd+Shift+R` (Mac) 硬刷新
- 或右键刷新按钮 → "清空缓存并硬性重新加载"

## ✅ 验证检查

### 后端验证

1. **健康检查**
   - 访问：`http://localhost:5000/health`
   - 应该返回：`{"status": "healthy"}`

2. **API 端点测试**
   - 访问：`http://localhost:5000/api/domain`
   - 应该返回：`[]`（空数组，如果没有数据）

### 前端验证

1. **浏览器控制台**
   - 打开开发者工具（F12）
   - 查看 Console 标签
   - 不应该有 405 或 CORS 错误

2. **网络请求**
   - 打开开发者工具（F12）
   - 查看 Network 标签
   - 刷新页面
   - `/api/domain` 请求应该返回 200 状态码

## 🔧 常见问题

### 问题 1: 后端无法启动

**错误**：`Address already in use`

**解决**：
```bash
# Windows
netstat -ano | findstr :5000
taskkill /PID <PID号> /F

# Mac/Linux
lsof -ti:5000 | xargs kill -9
```

### 问题 2: 前端无法连接后端

**检查**：
1. 后端是否正在运行？
2. 访问 `http://localhost:5000/health` 是否正常？
3. 前端控制台是否有错误信息？

### 问题 3: 仍然出现 405 错误

**解决**：
1. **必须重启后端服务**（重要！）
2. 清除浏览器缓存
3. 检查后端控制台是否有路由注册信息

## 📝 重要提示

1. **后端必须先启动**：前端依赖后端 API
2. **必须重启后端**：修改路由后必须重启才能生效
3. **清除浏览器缓存**：确保使用最新的前端代码
4. **检查控制台**：查看详细的错误信息

## 🎯 成功标志

当一切正常时，你应该看到：

1. ✅ 后端控制台显示：`Running on http://0.0.0.0:5000`
2. ✅ 前端页面正常加载，没有错误
3. ✅ 浏览器控制台没有 405 或 CORS 错误
4. ✅ Network 标签中 API 请求返回 200 状态码
5. ✅ 首页能正常显示统计数据（即使为 0）
